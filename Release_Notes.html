Purpose

BLE_Basic_DataThroughput_Server example demonstrates how to use data throughput via Notifications from GATT Server to GATT Client using the BLE component. This is similar functionliaty as the BLE_DataThroughput example from STM32CubeWB, only this example includes only the basic user code for performing the data transfers to the Client via Notifications (does not support Client functionality). It also includes all the configuration per section 7.6.8 of AN5289 Rev. 6 to maximize data throughput.

BLE_Basic_DataThroughput_Server example is being delivered as-is, not necessarily supported by ST.

This example includes the HAL and LL drivers provided in the STM32CubeWB_v1.13.1 package. For further information about these components, see official package.

Update History

Main Changes

Initial release of BLE_Basic_DataThroughput_Server Example

In the BLE_Basic_DataThroughput_Server example, the project is generated with the STM32CubeMX tool to initialize the system, peripherals and middleware stacks. User can open the provided ioc file in STM32CubeMX to modify the settings and add additional peripherals, to customize his final application, or generate project for different compiler/toolchain; i.e. MDK-ARM, STM32CubeIDE.
